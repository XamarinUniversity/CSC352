Before you begin

This is a Xamarin.Forms class. To complete the coding exercises, you will need a Xamarin.Forms development environment with either an emulator or a device to run the exercise on. Please see the setup page if you need help installing the Xamarin development environment.

Overview

In this class you will learn how to create and use multiple threads to improve application performance and responsiveness.

Objectives

  1. Thinking about Thread Safety
  2. Protecting multi-step operations

Lecture

CSC352 - Thread safety and Synchronization.

Exercises

Exercise materials are available from GitHub. The code can be cloned through a desktop Git client or downloaded directly as a ZIP file.

  1. Use a monitor lock to control access to a resource
    Exercise
    In this exercise, we will use locks to manage access to shared objects using multiple threads.
  2. Using the lock statement in place of Monitor.Enter
    Exercise
    In this exercise, we will replace our use of Monitor.Enter and Monitor.Exit with the C# lock keyword.
  3. Refine the lock usage to reduce contention
    Exercise
    In this exercise, we will move to a lock for each cat to optimize the contention.
  4. Using the lock ordering technique to remove potential deadlocks
    Exercise
    In this final exercise, we will use lock-ordering to ensure no deadlocks occur by acquiring multiple monitors.
